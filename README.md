# Arctic SAR Super Resolution

Super resolution спутниковых снимков льдов Арктики в радиоспектре (SAR) на основе каскада моделей Real-ESRGAN и веб-интерфейса на FastAPI.

Проект решает задачу увеличения пространственного разрешения низкокачественных SAR‑снимков льдов Арктики с сохранением текстуры и ледовых структур, чтобы улучшить последующий анализ (сегментацию, классификацию, мониторинг ледовой обстановки).

---

## Основная идея

- Исходные данные: одноканальные SAR‑патчи льдов Арктики (низкое разрешение).
- Модели: три последовательно применяемые генеративные модели (модифицированный Real‑ESRGAN) для увеличения:
  - `32×32 → 64×64`
  - `64×64 → 128×128`
  - `128×128 → 256×256`
- Итоговый масштаб: увеличение разрешения по каждой стороне в 8 раз относительно исходного патча (32 → 256).

Web‑приложение позволяет загрузить один или несколько снимков (PNG, JPG, TIFF), произвести супер‑разрешение с учётом выбранного режима нарезки на патчи, наблюдать процесс (быстрый предпросмотр) и скачать результаты в удобных форматах.

## Возможности

### Модели и пайплайн

- Три обученных генератора `Real-ESRGAN`‑типа (RRDB‑архитектура без BatchNorm, perceptual + adversarial loss, VGG‑перцептуальные признаки основаны на стандартной реализации VGG19).[web:68][web:69][web:71]
- Поддержка двух схем обработки:
  1. **Каскад с перенарезкой**  
     - Изображение последовательно увеличивается:
       - разрезается на патчи `32×32` (нахлёст 8), обрабатывается моделью `32→64`, склеивается;
       - полученный результат разрезается на `64×64` (нахлёст 16), обрабатывается моделью `64→128`, склеивается;
       - далее `128×128` (нахлёст 32), модель `128→256`, склейка.
  2. **Каскад без перенарезки между стадиями**  
     - Изображение один раз разрезается на патчи `32×32` (нахлёст 8);
     - каждый патч прогоняется через все три модели последовательно (`32→64→128→256`);
     - склейка выполняется один раз по финальным патчам `256×256`.
- Сшивка патчей с overlap‑областями через **взвешенное усреднение** (Hann‑маска), что снижает артефакты на границах.

### Веб‑приложение

- Backend на **FastAPI**:
  - эндпоинт `POST /start` принимает несколько файлов и опции обработки;
  - фоновые задачки (`BackgroundTasks`) выполняют расчёт на CPU/GPU;
  - эндпоинт `GET /task/{task_id}` даёт статус и данные для прогресса/превью;
  - эндпоинты скачивания:
    - `GET /download/{task_id}/file/{filename}/{format}` — отдельный файл в `png` / `jpg` / `tiff`;
    - `GET /download/{task_id}/all/{mode}` — ZIP со всеми результатами:
      - `mode=original` — в исходных форматах (или ближайшем доступном);
      - `png`, `jpg`, `tiff`.
- Frontend — минималистичный интерфейс (HTML + Tailwind CSS):
  - загрузка нескольких файлов разных форматов;
  - чекбокс **быстрого отображения** (по мере обработки патчей собирается и обновляется превью);
  - чекбокс **перенарезки между стадиями** (выбор режима пайплайна);
  - прогресс‑бар по задаче и статус;
  - превью исходных и улучшенных изображений;
  - кнопки скачивания по каждому файлу и «скачать всё одним архивом».

### Работа с форматами и метаданными

- Входные форматы:
  - `PNG`, `JPG/JPEG`, `TIFF` (в т.ч. GeoTIFF).
- Внутренняя обработка:
  - все изображения приводятся к одноканальному `L` (grayscale);
  - нормализация в диапазон \([-1, 1]\) для подачи в PyTorch‑модели.
- Выходные форматы:
  - `PNG` и `JPG` — обычное сохранение с управляемыми параметрами;
  - `TIFF` — при наличии исходных TIFF‑метаданных используется `tiffinfo`/`tag_v2`, что позволяет сохранить основные теги и служебную информацию.[web:61][web:31][web:34]

```
├── schemas/

├── static/
│   └── index.html
├── main.py
├── requirements.txt
├── README.md
├── realesrgan_32_to_64.py
└── true_patch.py
```

### Назначение основных файлов

- `main.py` — FastAPI backend, маршруты API, запуск задач обработки изображений.
- `services/sr_model.py` — загрузка моделей, архитектура генератора, инференс.
- `schemas/__init__.py` — Pydantic-схемы запросов и ответов.
- `static/index.html` — пользовательский интерфейс для загрузки и обработки снимков.
- `models/` — веса трёх обученных моделей super resolution.
- `realesrgan_32_to_64.py` — скрипт обучения/экспериментов для модели 32→64.
- `true_patch.py` — вспомогательный скрипт для работы с патчами.

## Установка и запуск

### 1. Клонирование

```
git clone https://github.com/<username>/arctic-sar-super-resolution.git
cd arctic-sar-super-resolution
```

### 2. Установка зависимостей

Рекомендуется Python 3.10+ и виртуальное окружение.

```
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

pip install --upgrade pip
pip install -r requirements.txt
```

### 3. Модели

Поместите обученные веса в папку `models/`:

```
models/
├── model_32_64.pth
├── model_64_128.pth
└── model_128_256.pth
```

Имена файлов должны совпадать с указанными в `main.py`. Веса соответствуют архитектуре `Generator` из `services/sr_model.py` (RRDB‑генератор без BatchNorm и с PixelShuffle×2 на выходе).

### 4. Запуск FastAPI сервера

```
uvicorn main:app --reload
```

После запуска приложение будет доступно по адресу:

```
http://127.0.0.1:8000
```

## Использование

1. Откройте веб‑страницу `http://127.0.0.1:8000`.
2. Загрузите один или несколько спутниковых снимков:
   - поддерживаются `PNG`, `JPG/JPEG`, `TIFF`;
   - можно смешивать форматы в одной задаче.
3. Выберите опции:
   - **Быстрое отображение** — по мере обработки патчей будет обновляться превью собранного изображения;
   - **Перенарезка между стадиями** — включить/отключить промежуточный re‑tiling между тремя моделями.
4. Нажмите «Начать улучшение»:
   - в блоке «Статус» появится прогресс;
   - справа в «Результаты» будут показаны исходные и улучшенные превью.
5. После завершения:
   - можно скачать результат отдельно для каждого файла в форматах `PNG`/`JPG` и при исходном `TIFF` — также `TIFF`;
   - можно скачать ZIP‑архив со всеми результатами:
     - в исходных форматах (`/all/original`);
     - либо принудительно в `png` / `jpg` / `tiff`.

## Обучение моделей

Скрипт обучения реализует:

- датасет с чтением патчей LR/HR из сетевой SMB‑шары;
- генератор на основе RRDB‑блоков;
- U‑Net‑подобный дискриминатор со спектральной нормализацией;
- перцептуальную потерю (VGG19 features) и GAN‑часть;
- EMA‑усреднение весов генератора;
- метрики PSNR и ENL (Equivalent Number of Looks) для оценки качества.

Обучение каждой из трёх моделей (32→64, 64→128, 128→256) проводится отдельно с аналогичной архитектурой, но на соответствующих размерах патчей. Веса сохраняются как `*.pth` и затем используются в инференсе.

## Ограничения и планы

Текущие ограничения:

- Основная оптимизация ориентирована на **одноканальные SAR‑данные** (grayscale).
- Память задач и результаты хранятся в памяти приложения (при рестарте сервера задачи теряются).
- Живой прогресс реализован через polling (регулярные запросы к `/task/{task_id}`); WebSocket‑канал пока не подключён.

Планы развития:

- вынести хранение задач и результатов на диск / в БД;
- добавить WebSocket или Server‑Sent Events для более отзывчивого прогресса и предпросмотра;
- улучшить поддержку GeoTIFF (сохранение полной геопривязки, CRS, transform через специализированные библиотеки);
- добавить метрики качества (PSNR/SSIM/LPIPS) для пользовательского сравнения с baseline‑методами;
- интегрировать сервис в более широкий пайплайн анализа ледовой обстановки.
